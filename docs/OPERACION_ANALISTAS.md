# Manual rápido para analistas

## Flujo de atención

1. Revisar tickets nuevos o asignados.
2. Validar severidad y caso de uso.
3. Asignar responsable si aplica.
4. Cambiar estado a `En Progreso`.
5. Registrar avances con comentarios públicos o notas internas.
6. Adjuntar evidencias en el comentario correspondiente.
7. Usar `En Espera` solo cuando falte información o tercero.
8. Resolver con comentario de cierre técnico.
9. Cerrar cuando el caso esté validado.

## Uso de evidencias

Cada comentario puede tener sus propios adjuntos. Esto permite mantener el hilo del incidente ordenado.

## Notas internas

Las notas internas no son visibles para solicitantes.
